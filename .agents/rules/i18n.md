# Internationalization (i18n) Standards

This document defines the mandatory architectural and implementation standards for all internationalization (i18n) related code in the Trevvo design system and applications.

## 🏗️ Framework & Tools
- **Core:** Use `@ngx-translate/core` for Angular applications.
- **Storage:** Translation keys are stored in JSON files located in `design-system/i18n/` (e.g., `en-GB.json`, `pt-PT.json`, `es-ES.json`).
- **Keys Hierarchy:** Use a structured, nested JSON hierarchy:
  - `atoms.<componentName>.<key>`
  - `molecules.<componentName>.<key>`
  - `organisms.<componentName>.<key>`
  - `templates.<componentName>.<key>`
  - `common.<key>` for global reusable strings.

## ⚙️ Implementation in Components
- **Zero Hardcoded Strings:** NEVER use literal strings for UI text in components or templates.
- **Templates:** Use the `translate` pipe: `{{ 'key' | translate }}`.
- **Component Inputs:**
  - Pass translation keys (as strings) to child components' `label`, `placeholder`, `hint`, etc.
  - Child components must use the `translate` pipe internally to render these inputs.
- **Error Messages:** Methods returning validation errors must return translation keys (e.g., `organisms.form.errors.required`).

## 🔋 Form Buttons Loading Pattern
When implementing forms with submission states, always use localized keys for buttons:
- `saveBtn`: For the default state.
- `loadingBtn`: For the processing state.
- `cancelBtn`: For the cancel action.

**Example Template Pattern:**
```html
<ds-button (click)="submit()" [disabled]="invalid || isSubmitting()">
  @if (isSubmitting()) {
    <div class="flex items-center gap-3">
      <div class="w-5 h-5 border-2 border-white/30 border-t-white rounded-full animate-spin"></div>
      <span>{{ 'organisms.form.loadingBtn' | translate }}</span>
    </div>
  } @else {
    {{ 'organisms.form.saveBtn' | translate }}
  }
</ds-button>
```

## 🧪 Testing & Storybook
- **Previews:** Storybook stories must be able to resolve translation keys. While global configurations should handle this, ensure individual stories are i18n-aware if needed.
- **Tests:** Verify that components correctly handle and render translation keys.

## 📖 Key Management
- Every new key added to a component MUST be added to ALL supported locale JSON files.
- Maintain consistency in terminology across different languages.
